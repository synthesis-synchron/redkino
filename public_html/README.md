# Деплой на reg.ru (обычный хостинг)

Папка `public_html/` — готовая статическая сборка сайта. Залейте её содержимое в корневую папку сайта на хостинге reg.ru (обычно `public_html` или `www`).

## Что внутри
- `index.html` — главная страница, уже отрендеренная (виден весь контент даже без JS, хорошо для SEO).
- `404.html` — страница 404 (та же главная, чтобы любые URL открывались).
- `assets/` — JS, CSS, изображения с хешами в имени (кэшируются навсегда).
- `lovable-assets/` — вспомогательные ассеты.
- `.htaccess` — настройки Apache: SPA-роутинг, сжатие, кэш, MIME-типы.

## Как залить через FTP / Файловый менеджер reg.ru
1. Откройте панель reg.ru → раздел «Хостинг» → «Файловый менеджер» (или подключитесь по FTP).
2. Зайдите в папку сайта (обычно `public_html` или `www/<домен>`).
3. **Удалите** старый `index.html` / тестовую заглушку хостера.
4. Загрузите **всё содержимое** папки `public_html/` (включая скрытый `.htaccess`).
5. Откройте сайт в браузере — должно открыться без ошибок.

## Как получить ZIP для загрузки
```
cd public_html && zip -r ../site.zip . .htaccess
```
Полученный `site.zip` распакуйте на хостинге через файловый менеджер reg.ru.

## Обновление сайта
После любых правок в Lovable заново нажмите «Собери статическую сборку» — папка `public_html/` пересоберётся, перезалейте её на хостинг.
